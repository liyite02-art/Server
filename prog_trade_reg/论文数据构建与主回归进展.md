## 2. 数据是如何生成的

整体逻辑是：**高频（逐笔 + LOB）在单日内尽量各读一遍 → 聚合为股票×日 → 再与政策前暴露合并为长表**。

### 2.1 原始输入

- **逐笔成交 `trans_fea`**：按交易日一个 `.fea` 文件；代码中用 `pandas.read_feather` 经 `feather_io` 转为 Polars。成交价按 `TRADE_PRICE_SCALE`（默认 `10000`）缩放到与 LOB 一致的数量级。
- **LOB**：按交易所（SZ/SH）与 `trade_date` 定位 Parquet；经 `lob_agg.aggregate_lob_by_code_time` 在 `(code, time)` 上形成快照（含买一卖一等），供与成交对齐。

### 2.2 日度因变量：`daily/outcomes/YYYYMMDD.parquet`

**生成命令**：`python -m prog_trade_reg outcomes <YYYYMMDD>` 或批量 `outcomes-range` / `build-all` 中的因变量段。

**代码入口**：`prog_trade_reg/pipeline/outcome_daily.py` → `build_outcome_daily_for_date`。

**单日流程（每个交易所一轮）**：

1. 读取当日 `trans_fea`，筛出带 `.SZ` / `.SH` 后缀的代码，解析为整数 `code`，保留 `time`、`tradePrice`、`tradeVolume`，过滤价量为正。
2. 读取该所 LOB，得到按 `(code, time)` 排序的快照。
3. **有效价差（VWES 的分子/分母）**：逐笔成交 `join_asof` 到同代码上**不晚于**成交时间的最近 LOB；用 `mid = (bp1+sp1)/2`，逐笔有效价差（基点）`es_bps = 2|price−mid|/mid×10000`；再按股票聚合  
   `es_bps_num = Σ(es_bps × vol)`，`es_bps_den = Σvol`。日度成交量加权有效价差为 **`vw_es_bps = es_bps_num / es_bps_den`**（在长表 `panel-did` 中写出）。
4. **`rv_mid`**：由 LOB 中间价序列的对数差分平方在日内求和（实现波动率）。
5. **`amihud`**：基于日内首末中间价对数差的绝对值，除以当日美元成交额（与 ES 聚合中的 `dollar_vol` 衔接）。

**输出表主键**：`(exchange, code, trade_date)`；列包括 `n_trades`、`volume_sum`、`dollar_vol`、`es_bps_num`、`es_bps_den`、`rv_mid`、`amihud` 等（见 README 摘要）。

### 2.3 政策前连续暴露：`meta/exposure_pre_policy.parquet`

**生成命令**：`python -m prog_trade_reg exposure`（可指定 `--start` / `--end` / `--trade-days-pkl`）。

**代码入口**：`prog_trade_reg/pipeline/exposure.py` → `build_exposure_pre_policy`。

**逻辑**：

1. 在窗口内按**交易日历**遍历每个 `trade_date`，每个交易日对 `trans_fea` **只读一次**，在 `compute_daily_trans_exposure_features` 中：
   - 仅保留 `bsFlag ∈ {B,S}` 的成交，按 `(exchange, code)` 聚合买/卖量；
   - **日度 OIB**：`oib_day = (buy_vol − sell_vol) / (buy_vol + sell_vol)`；
   - 同时得到 `n_trades_day`、`total_vol_day` 等。
2. 将多日面板在 `(exchange, code)` 上再做**时间维度平均**（`oib_day` → `oib_mean_raw` 等），得到每只股票在政策前窗口上的**标量暴露**。
3. 对三个原始均值列分别做**截面 winsor**（默认 1%–99%）得到 `*_mean`，再在全截面 **z 标准化** 得到 **`E_oib_z`、`E_n_trades_z`、`E_total_vol_z`**。  
   **主回归使用的连续暴露强度**对应 **`E_oib_z`**（订单流不平衡）；后两者可作控制或扩展。

**重要性质**：暴露对每只股票**不随日历日变化**（政策前后 merge 到长表上为同一值）；与施行日虚拟 `post` 的乘积 **`post × E_oib_z`** 在政策前恒为 0，政策后为 `E_oib_z`，跨股票有横截面差异。

### 2.4 DID 长表：`panel/panel_did_long.parquet` 与 `.csv`

**生成命令**：`python -m prog_trade_reg panel-did`（需已有 `daily/outcomes/*.parquet` 与 `meta/exposure_pre_policy.parquet`）。

**代码入口**：`prog_trade_reg/pipeline/build_panel.py` → `build_panel_did_long`。

**逻辑**：

1. 扫描 `daily/outcomes/` 下所有日度 Parquet，纵向拼接为长表。
2. 左连接 `exposure_pre_policy.parquet`，键为 `(exchange, code)`。
3. 定义 **`post = 1{trade_date ≥ POLICY_DATE_MAIN}`**（可用 CLI `--policy-date` 覆盖）。
4. 对每个暴露 z 列生成 **`post_x_{列名}`**（代码中即 **`post_x_E_oib_z = post × E_oib_z`** 等）。
5. 构造 **`panel_unit`**（如 `SZ_000001` 形式）并计算 **`vw_es_bps`**（由 `es_bps_num` / `es_bps_den`）。

**CSV** 使用分隔符 `|`，便于 Stata 等导入。

## 3. 主回归如何跑

### 3.1 计量规格（与当前代码一致）

在 **股票实体 × 日历日** 面板上估计 **双向固定效应（TWFE）** 模型：

- **因变量（默认）**：`vw_es_bps`（成交量加权有效价差，基点）。
- **核心解释变量**：`post_x_E_oib_z`（连续 DID 交互项）。
- **固定效应**：`EntityEffects` + `TimeEffects`（`linearmodels` 中分别对应 `panel_unit` 与日历日）。
- **标准误（默认）**：按**时间**维度聚类（`cluster_time=True`）；可选 `--cluster twoway`（实体+时间双聚类）或 `--cluster none`（不建议用于正式推断）。

**实现**：`prog_trade_reg/did_regress.py` 中 `run_did_twfe`，使用 `linearmodels.panel.PanelOLS.from_formula`。

### 3.2 命令行

```bash
cd /path/to/prog_trade_reg
pip install -e .   # 依赖含 linearmodels>=5.3

# 全样本（读完整面板，适合正式结果）
python -m prog_trade_reg did-regress --cluster time
# 或双聚类
python -m prog_trade_reg did-regress --cluster twoway

# 快速子样本（默认 --head-strategy ends：CSV 取前/后各半段，避免仅政策前期导致 post×E 全为 0）
python -m prog_trade_reg did-regress --head 50000 --cluster time
```

常用可选参数：`--panel <path>` 指定面板文件；`--y` 换因变量（如 `rv_mid`、`amihud`）；`--x` 换交互列；`--control` 可重复传入额外控制列；`--head-strategy first|random|ends` 控制子样本方式。

### 3.3 与论文写作的对应关系

正文可表述为：在控制**股票 FE**与**交易日 FE**后，考察**政策施行后**与**事前连续暴露**交互项对流动性/价差类指标的影响；标准误按时间（或双向）聚类。TWFE 在异质性处理效应下的解释需结合文献（README §9 所列文献可作引用锚点）。

## 4. 初步结果（当前仅作方向性参考）

下列数字来自在本机面板 **`panel_did_long.csv`** 上的一次**快速子样本**回归（`--head 50000`、`--head-strategy ends`、按**时间聚类**），**不等同于全样本最终论文表**。全样本请运行不带 `--head` 的 `did-regress` 并固定于论文定稿规格。

| 变量 | 系数 | 聚类稳健标准误 | t 统计量 | 备注 |
|------|------|----------------|----------|------|
| `post_x_E_oib_z` | **−1.111** | 0.070 | ≈ **−15.9** | 对 `vw_es_bps`；子样本内 p 值接近 0 |

**解读（初步、非最终结论）**：在该子样本设定下，事前 OIB 暴露（z 分）与政策施行后的交互项与成交量加权有效价差呈**显著负向**关联，方向需结合经济含义与全样本、替代因变量、控制变量及稳健性检验再写定。

**全样本提示**：当前 `panel_did_long.csv` 规模较大（约数百万行），全样本回归对内存与时间要求更高；正式报告应以全样本（或预先设定的抽样规则）为准。

## 5. 复现流水线（最小顺序）

1. 配置 `config.py`（路径、政策日、暴露窗口、交易日历 `TRADE_DAYS_PKL` 等）。
2. `python -m prog_trade_reg exposure` → 生成 `meta/exposure_pre_policy.parquet`。
3. `python -m prog_trade_reg outcomes-range`（或 `build-all`）→ 生成 `daily/outcomes/*.parquet`。
4. `python -m prog_trade_reg panel-did` → 生成 `panel/panel_did_long.*`。
5. `python -m prog_trade_reg did-regress` → 估计主规格并阅读输出表。
